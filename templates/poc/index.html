Vue d'ensemble

Mode sandbox

Prendre en main la sandbox

Chaque page simule un scénario clé du produit afin de tester rapidement l'API sans construire le front-end complet. Utilisez la page Connexion pour obtenir des tokens JWT puis naviguez vers les autres étapes.

Itinéraires conseillés

  1. Collaborateur : Inscription → Connexion → Création d'organisation → Explorer → Détail athlète → Messagerie.
  2. Agent : Inscription → Connexion → Création d'athlète → Messagerie.
  3. Collaborateur invité : Connexion → Rejoindre une organisation (via invitation) → Explorer → Suivre un athlète.

Conseils rapides

  • Les appels API sont réalisés via fetch et les réponses sont affichées au format JSON.
  • Vous pouvez copier l'access token depuis la barre latérale pour tester directement dans des outils externes.
  • Le bouton « Effacer les tokens » permet de simuler une déconnexion.

Raccourcis utiles

Endpoints d'authentification

  • POST /api/users/register/
  • POST /api/users/login/
  • POST /api/users/refresh/

Pour démarrer avec un collaborateur

  • POST /api/organisations/
  • GET /api/athletes/
  • POST /api/follows/athletes/<id>/follow/

Pour les agents

  • GET /api/sports/
  • POST /api/athletes/
  • GET /api/messaging/threads/

Messagerie

  • POST /api/messaging/threads/
  • GET /api/messaging/threads/<id>/messages/
  • POST /api/messaging/threads/<id>/messages/
